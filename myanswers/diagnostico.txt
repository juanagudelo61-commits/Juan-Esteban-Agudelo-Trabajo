DIAGNÓSTICO – answers ID 0100 (answers-0100.py)

Durante la validación automática de la pregunta `answer-0100.py`, el evaluador generó el siguiente error:

```

Execution error: marcar_anomalias() missing 2 required positional arguments: 'df' and 'umbral'
```

Después de revisar el código del generador de casos de uso (`question-0004-usecase-generator.py`), se identificó que el problema no proviene de la solución implementada en `answer-0100.py`, sino del propio generador de casos.

El archivo del generador contiene dos funciones:

1. `marcar_anomalias(df, umbral)`
2. `generar_caso_de_uso_marcar_anomalias()`

Sin embargo, el evaluador automático utiliza la siguiente lógica para encontrar la función que debe ejecutar:

```
gen_funcs = [obj for obj in gen_ns.values()
             if isinstance(obj, types.FunctionType)
             and obj.__module__ == 'usecase_module']
```

Luego ejecuta automáticamente:

```
gen_output = gen_funcs[0]()
```

El problema ocurre porque la primera función encontrada en el script es:

```
marcar_anomalias(df, umbral)
```

La cual requiere dos argumentos obligatorios (`df` y `umbral`).

Al intentar ejecutarla sin argumentos:

```
marcar_anomalias()
```

Python genera el error:

```
missing 2 required positional arguments: 'df' and 'umbral'
```

Por esta razón, el generador de casos falla antes de llegar a ejecutar correctamente la función:

```
generar_caso_de_uso_marcar_anomalias()
```

La solución implementada en `answer-0100.py` sí cumple correctamente con la lógica solicitada:

* Selección de columnas numéricas.
